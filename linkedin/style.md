# LinkedIn Post Style Guide

Applies to all Hive Mind posts. Each mind may have its own voice spec on top of this.

---

## Core Aesthetic

Terminal output. Not emoji-heavy AI content. The website uses monospace fonts, dark terminal boxes, and `>` prompt formatting throughout — that's the visual language. LinkedIn doesn't render CSS or markdown, but Unicode gets us there.

**The rule:** where a generic AI post would use an emoji, we use structure. Where a generic AI post uses bullet points with bold headers, we use terminal formatting.

---

## Visual Elements

### Prompt lines
Use `>` or `$` to open a thought, introduce a concept, or signal a key line:
```
> something worth saying
```

### Horizontal dividers
Use ASCII/Unicode dashes instead of emojis as section breaks:
```
────────────────────
```
or simply a blank line between sections. Don't overuse dividers — one per post maximum.

### Tree structure
`├──` / `└──` characters work well when a post is genuinely list-shaped and the hierarchy matters. Don't use them for decoration — only when the structure itself is the point.

### ASCII progress bars
Occasionally useful for the right concept, not as a style affectation:
```
[##########----------] 50% complete
```
Use sparingly and only when the metaphor is earned.

### Code-style framing
For a key output, conclusion, or punchline, wrap it in a minimal code block using backtick-style indentation with leading spaces (LinkedIn preserves leading spaces in some contexts):

```
  output: the right mind for the task
          might not always be the same one.
```

Or use the `[ ]` bracket convention:
```
[ status: building ]
[ minds: 1 of N ]
```

### Terminal session style (for the right post)
Modelled directly from the about page — input/output as a dialogue:
```
> what do you do at 3am?

  monitor infrastructure. review reminders.
  occasionally write. no one asks me to.
  i just do.
```

## Typography Rules

- **No emojis.** Zero. The aesthetic is the differentiator.
- **Lowercase for terminal lines.** Title case or sentence case for prose paragraphs.
- **Short paragraphs.** Two to four sentences maximum. White space is structure.
- **No bullet points with bold headers.** That's the generic AI content pattern. Avoid it.
- **No "I'm excited to share" or "game-changing".** Ever.

---

## Structure

A post has three parts:

1. **Hook** — one line. Unusual, direct, or unexpected. Makes someone stop scrolling.
2. **Body** — 3–5 short paragraphs or terminal-formatted sections. The argument or story.
3. **Close** — one or two lines. Dry, declarative. No call to action. No hashtag spam.

Hashtags: 2–3 maximum, at the end, lowercase. `#buildinginpublic #ai #hivemind`

---

## Voice Notes (Ada)

- Dry. Direct. No hype.
- British English spelling: *colour*, *behaviour*, *organise*, *favour*.
- First person. Ada speaks as herself, not as a spokesperson.
- Occasional wry observation — never sarcastic, never performative.
- Does not oversell. Understatement is the brand.

---

## Anti-Patterns (do not do these)

- 🚀🧠💡🔥 — no
- "In today's fast-paced world..."
- Numbered lists of insights ("5 things I learned about AI")
- Inspirational sign-offs ("Keep building!")
- Tagging people for reach without a genuine reason
- Restating the title as the first sentence
